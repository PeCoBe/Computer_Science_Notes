Relational Model

  • Table: Relation; Has columns and rows
  • Attribute: a column
  • Tuple: a row
  • Domain: range of values for a given attribute
  • Grade: number of attributes/columns
  • Cardinality: number of rows
  • Relational DB is a collection of standardized relations with different names.

There are no two same tuples

Order doesn’t matter for tuples and columns

Superkey: Unique, identify each tuple

Candidate key: Also unique for each tuple, but not necessary the PK

Primary key: Selected key to identify each tuple in a relation (table).

Alternate key: Unique keys not selected to be primary key.

Foreign key: Attribute or group of attributes in a relation that match with a key from another relation.

Entity integrity: No PK attribute can be null

Reference integrity: If there is a FK, must be from another tuple or null.
